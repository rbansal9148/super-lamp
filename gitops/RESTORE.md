# Disaster Recovery — restore runbook

Single-node k3s + ArgoCD GitOps. This is the path back after node/disk loss.
Read it top to bottom; step 0 is the one that, if skipped, makes everything else
impossible.

---

## 0. The one backup you MUST have off-cluster: the SealedSecrets key

Every `*.sealedsecret.yaml` in this repo is encrypted to the SealedSecrets
controller's private key. **If that key is lost, every committed secret is
permanently undecryptable** — a perfect git repo cannot bootstrap, and every DB
password / API token / Authelia signing key must be regenerated by hand.

The key is NOT in git (correctly). Back it up off-cluster, now and after any key
rotation:

```sh
# Dumps the controller's sealing keypair(s). Store the output in an OFFLINE,
# encrypted location (password manager / encrypted USB / age-encrypted file).
# NEVER commit this.
kubectl -n kube-system get secret \
  -l sealedsecrets.bitnami.com/sealed-secrets-key \
  -o yaml > sealed-secrets-key.backup.yaml
```

Verify the backup is non-empty and contains `kind: Secret` with `tls.key` data.
That's the asymmetric-risk item: costs one command today, costs everything if
missing after node loss.

## 1. Data backups (scoped — see below)

Only the NON-reproducible data needs off-node backup; the scraper DBs re-scrape.

| Data | Why back up | Location |
|---|---|---|
| Authelia DB (users, TOTP enrolments) | TOTP secrets cannot be re-derived | PVC `authelia-data` (local-path-retain) |
| Prowlarr config + DB | hours of hand-configured indexers + API keys | hostPath `/opt/docker/data/prowlarr` |
| bitmagnet DB | ~weeks of DHT crawl, slow to rebuild | hostPath `/opt/docker/data/bitmagnet/db` (43GB) |
| Calibre library | irreplaceable user data (books) | hostPath `/opt/docker/data/calibre-web-automated` |

Reproducible (do NOT need backup — re-scrape on restore): zilean, comet,
aiostreams, aiometadata, stremthru DBs and all redis (caches).

> A `pg_dump`/tar CronJob to off-box storage for the four rows above is a
> recommended follow-up not yet implemented. Until then, snapshot the hostPath
> dirs + the retain PVCs manually before any risky change.

## 2. Restore order (node rebuilt from scratch)

1. **Install k3s** (same version; this cluster ran v1.35.x). Restore
   `/opt/docker/data/*` hostPath dirs from backup BEFORE starting workloads —
   the postgres/calibre/prowlarr/stremthru/bitmagnet pods mount `type: Directory`
   and won't start if the dir is missing.
2. **Install ArgoCD** into the `argocd` namespace.
3. **Restore the SealedSecrets controller key** from step 0 BEFORE the controller
   first generates a fresh one:
   ```sh
   kubectl apply -f sealed-secrets-key.backup.yaml
   # then let the sealed-secrets controller install (step 4) and adopt it,
   # or restart the controller so it picks up the restored key.
   ```
4. **Bootstrap everything** — one command brings up the whole cluster:
   ```sh
   kubectl apply -f gitops/bootstrap/
   ```
   This applies three root objects:
   - `sealed-secrets.yaml` — the controller (sync-wave -2, before any Secret).
   - `root-app.yaml` — app-of-apps for the **observability** stack.
   - `apps-root.yaml` — app-of-apps for **everything else** (cert-manager + the
     ~21 streaming/infra apps). Recurses `gitops/apps/`, excludes
     `observability/**` (owned by root-app).
5. ArgoCD reconciles the rest: cert-manager issues the wildcard LE cert,
   external-dns repopulates Cloudflare DNS, the apps come up against their
   restored data.

## 3. Bootstrap prerequisites (out-of-band, not in git)

- **cert-manager controller** — installed out-of-band; `gitops/apps/cert-manager.yaml`
  ships only the issuer + token + wildcard cert CRs that depend on it.
- **ArgoCD repository credential** — the SSH deploy key for
  `git@github.com:rbansal9148/super-lamp.git` (Secret labelled
  `argocd.argoproj.io/secret-type=repository`). Back this up alongside the
  sealing key.
- The ClusterIssuer is named **`letsencrypt`** (DNS-01 via the Cloudflare token).
- **Prowlarr indexer config is DB-only (not declarative, not in git).** Indexers and
  indexer-proxies live in prowlarr's DB under the `/opt/docker/data/prowlarr` hostPath
  (StorageClass `local-path-retain`). Restoring that hostPath restores them; a *clean*
  rebuild without it means re-adding indexers via the Prowlarr UI/API. Two fixups that
  must hold after any rebuild: the Bitmagnet (Local) indexer URL is
  `http://bitmagnet:3333/torznab` (NOT the legacy `bitmagnet_vpn`), and the "Http"
  indexer-proxy points at gluetun port **8888**. prowlarr also needs `bitmagnet` in its
  `NO_PROXY` (in git) so internal torznab fetches bypass the VPN proxy. API key:
  `<ApiKey>` in `/opt/docker/data/prowlarr/config.xml`.

---

## 4. What single-node does NOT protect against

Node/disk loss is total outage — there is no HA on one node, by design. The
mitigation is off-node backups (steps 0–1), which convert "permanent loss" into
"rebuild + restore." HA is explicitly out of scope for this homelab.
